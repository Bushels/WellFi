# WellFi Hero Lessons Learned

This file preserves the useful conclusions from the exploratory hero work.

> **2026-03-21 override:** The `tool-only` conclusion below is no longer the full rule. For this marketing portion, the winning direction was the **approved side-clamped installed-state render** on a black background, treated as the visual source of truth. The useful lesson from the old file is restraint, not the removal of the pipe reference at all costs.

## Updated Lesson

- `Exact approved render` beats `technically available model`.
- `Black industrial restraint` beats `clever cinematic transitions`.
- `R3F as exact-match animation container` beats `pretending the raw GLB is hero-ready`.
- If the still frame does not already match the approved marketing image, no amount of animation will save it.

## Decision

For the first-load hero, a `tool-only product stage` is the better direction.

The `pipe-mounted explainer` is still useful, but it belongs lower on the page where the user has time to process installation mechanics.

## Why The Pipe-Mounted Hero Drifted

- It turned the first impression into an explainer instead of a product entrance.
- The hardware story became too dense for a `2 to 3 second` intro.
- The scene started optimizing for installation logic instead of visual impact.
- The added pipe, coupling, and clamp geometry diluted the product silhouette.
- Mobile composition became harder because the mechanism demanded too much screen space.

## What Still Matters From That Work

- The product must not look like generic AI art.
- Dark cinematic lighting is the right direction.
- Cyan signal motion works only if it is restrained and tied to a real purpose.
- Material separation matters:
  - black pipe if shown
  - stainless tool body
  - realistic fiberglass if shown
- Mobile must be composed deliberately, not treated as a scaled desktop view.

## Tool-Only Hero Rules

- Show the WellFi tool as the hero object.
- Let the product silhouette carry the first read.
- Keep one subtle signal cue so the user understands this is telemetry hardware.
- Use installation language later on the page, not in the first visual hit.
- Preserve scale with lighting, shadows, and camera, since the pipe reference is gone.

## Copy Lessons

- First-load copy should lead with capability, not installation steps.
- Product hero copy should focus on:
  - wireless telemetry
  - pressure at surface
  - production tuning
- Retrofit/changeout language belongs in the workflow and proof sections.

## Gemini Critique Summary

Gemini was used as a narrow critic for the pivot.

Useful takeaways:

- `Tool-only` is stronger for first-load impact because it anchors attention on the real hardware.
- Keep:
  - dark cinematic lighting
  - metallic reflections
  - subtle industrial atmosphere
- Shift the intro copy from installation to core capability.
- Main risk: a standalone cylindrical tool can lose scale and purpose, so the hero needs strong camera work, grounding shadows, and a clear signal cue.

## Next Use Of The Pipe-Mounted Story

If the pipe-mounted concept returns, use it in one of these places:

1. workflow section
2. scrollytelling explainer
3. technical installation page
4. case-study animation

Do not force it back into the first-load hero unless the product stage fails completely.
