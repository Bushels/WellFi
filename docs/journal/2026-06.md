## 2026-06-22 — Clearwater Descent (scroll parallax section)

Replaced the static `OneEightTrillionAnchor` ("1.8 trillion barrels") section with a
scroll-driven parallax **Clearwater Descent** on `/wellfi`.

Decisions (via brainstorming + visual companion):
- Concept A — continuous descent (one GSAP `ScrollTrigger` `scrub`, sticky stage, **no pinning**).
- Reveal 2 — isolated device on a cyan "data" glow; tagline **"Data Below, Insight Above"**; no CTA.
- Geology locked on **Clearwater** (not Bluesky) — more popular / futureproof play.
- Caption dropped (benefit-only descent); 6 benefits surface one at a time, surface→reservoir order:
  Pump Life → Production → Drawdown → Reservoir Monitoring → Water Cut → Well Optimization.
- Copy relocation: "Data Below, Insight Above" moved from the telemetry heading to the reveal;
  telemetry heading → **"Explore Your Data"**; Proof tagline → **"Decisions with Proof"**.
- Reliability over flash (prior parallax pain): no pinning, no image-scrubbing;
  `prefers-reduced-motion` + touch/narrow → static stacked fallback.

Architecture: pure, tested math in `src/lib/clearwater/descent.ts` (strata specs, benefit
visibility windows, reveal state); thin presentational components (`FormationStrata`,
`BenefitChip`, `DeviceReveal`); `ClearwaterDescent` drives styles via a single scrubbed
`onUpdate`. SSR renders the static layout; the client enhances to descent on
desktop + fine-pointer + motion (keyed on a `descent` state so the trigger measures the tall DOM).

Gotchas worth remembering:
- Site color tokens are **misnamed**: `--em-cyan` (#EF4444) and `--em-glow` (#F87171) are both
  RED. There is no blue token — the device "data" glow uses the literal `rgba(34,211,238,…)`,
  matching the cyan accent already on the telemetry cards.
- `next/image` does **not** auto-apply the `/wellfi` basePath in this repo; every image `src`
  hardcodes it. Caught a 404 in live preview and fixed `DeviceReveal` → `/wellfi/renders/...`.
- A subagent caught a real `-0` vs `0` bug in `layerTranslateVh` (Vitest `toBe` uses `Object.is`,
  and `Object.is(-0, 0) === false`); normalized with `+ 0`.

Verification: tsc/eslint clean, 57/57 vitest, `next build` ok, live preview (desktop + mobile).
Branch `feat/clearwater-descent`, 6 commits (`d026eb4`→`fc96c51`). Pending review/merge.

## 2026-06-23 — Clearwater render-based rebuild + hero WellFi fix (shipped to prod)

The flat-CSS "strata bands" Clearwater section (above) was **rejected by Kyle as generic**
("sewer pipe in a hole", not marketable). Rebuilt as a **Blender-rendered horizontal-well drilling
cutaway** and shipped to production (`mpsgroup.energy/wellfi`).

Clearwater (now live):

Hero WellFi fix:
- Kyle reported the hero WellFi had "moved outside the intermediate" + "animation not working". Evidence
  showed the hero geometry was byte-identical to the approved deploy and the animation was fine. Root
  cause: stray commit `f25fb2d` "lower below-pump WellFi placement" (`0.34→0.55`, dropping it to the
  casing shoe) rode live when `master` deployed. Reverted to **0.50** (`WELLFI_BELOW_PUMP_CASING_PARAM`,
  Kyle-dialed). Shipped `7ffd623`.

Lessons:
- **Deploying `master` ships every commit on it**, not just your change — review
  `git log <last-deployed>..master` before pushing. A stray hero tweak went live unreviewed this way.
- **Design quality:** real Blender renders, not flat CSS; don't bury design under heavy multi-agent
  process (burns tokens for weak output).
- **Blender via MCP:** long renders time out the MCP bridge → render **headless via CLI**
  (`blender --background file.blend --python script.py`). OIDN denoiser OOMs at 2560×1440 → use the
  OptiX denoiser. The Claude Blender bridge can wedge even with the addon listening on 9876 (restart it).
- `next/image` does not auto-apply the `/wellfi` basePath — hardcode it in every `src`.
- `feat/island-hero-v9` is a divergent **parked** hero, not the live one — don't restore it to fix the hero.
