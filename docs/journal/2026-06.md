Lessons:

  `git log <last-deployed>..master` before pushing. A stray hero tweak went live unreviewed this way.

  process (burns tokens for weak output).
- **Blender via MCP:** long renders time out the MCP bridge → render **headless via CLI**

  OptiX denoiser. The Claude Blender bridge can wedge even with the addon listening on 9876 (restart it).
- `next/image` does not auto-apply the `/wellfi` basePath — hardcode it in every `src`.
- `feat/island-hero-v9` is a divergent **parked** hero, not the live one — don't restore it to fix the hero.

Lessons:
- **"Commit" ≠ "deploy".** The auto-mode classifier blocked a `master` push authorized only as "commit this" — a
  prod deploy is a separate scope. Get an explicit "push/deploy" before pushing the site repo. (Asset-repo pushes don't deploy.)
- **AgX whitens hot emission.** A bright cyan emitter desaturates toward white under AgX — keep emission *low* for a
  saturated brand-cyan glow. The bloom comes from AgX rolloff, not raw strength.
- **mm-scale CAD needs camera `clip_end` raised** — the default 1000 clips a tool ~1400 mm from camera → empty frame.
- **Composite a render onto a card with transparent film + a CSS edge-fade mask**, not a flat dark bg (a flat bg always
  shows a rectangle; transparent composites seamlessly). WebP crushes dark transparent renders ~100× (2.9 MB → 25 KB).
- **Replace an unknown material rather than clamp it**: `WellFi_Collar_Candle` didn't expose emission via the inputs
  I checked, so forcing a fresh material gave control the clamp couldn't.
